## Anti-Patterns Verdict

LLM assessment: Not immediately AI-generated, but template-adjacent. Syne + DM Sans are both on the reflex-reject list. Every section has a mono uppercase eyebrow above its heading (named brand ban). Bento grid is well-executed but a recognized pattern. Hero-stat block reads as hero-metric template.

Deterministic scan: 5 findings, all layout-transition warnings. Lines 313 (cursor-ring), 444 (btn-ghost underline), 1702 (contact link underline), 1869 (loader-fill), 1908 (side-progress). All largely false positives -- underline animations run on 1px pseudo-elements; loader fires once at page load; side progress runs at 0.12s linear.

Browser overlay: detect.js injected into DOM but no findings reported back -- likely CSP-blocked. CLI signal only.

## Overall Impression

Technically sophisticated with one genuinely original element (the ID card) surrounded by recognizable patterns. The hero copy is strong. The problems are architectural: wrong fonts signal "template" before a word is read; section order follows a resume, not a persuasion flow; eyebrow labels repeat across every section like scaffolding never removed; certifications section actively undermines the credibility the research section builds. Biggest opportunity is information architecture.
